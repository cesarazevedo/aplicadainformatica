Aula 1: Introdução aos Algoritmos

O que são Algoritmos?

Um algoritmo é uma sequência de instruções realizadas com o objetivo de solucionar um problema. Em programação, utilizamos algoritmos para descrever as etapas que precisam ser efetuadas para que um programa execute as tarefas que lhe são designadas.

Características dos Algoritmos:

  • Conjunto de passos finitos e organizados
  • Quando executados, resolvem um determinado problema
  • São compostos por entrada (input), processamento e saída (output)

Pseudocódigo (Portugol)

O pseudocódigo é um método genérico de representar um algoritmo, usando uma linguagem simples sem necessidade de conhecer a sintaxe de linguagem de programação específica.

algoritmo "exemplo_basico" var numero: inteiro dobro: inteiro inicio leia(numero) dobro ← numero * 2 escreva("O dobro é: ", dobro) Fimalgoritmo

Estrutura Básica de um Algoritmo

Todo algoritmo em pseudocódigo segue uma estrutura padrão:

algoritmo "nome_do_algoritmo" // comentários e informações do autor var // declaração de variáveis inicio // comandos do programa Fimalgoritmo

Aula 2: Variáveis e Tipos de Dados

O que são Variáveis?

Variáveis são espaços alocados na memória do computador para armazenar dados ou expressões utilizados por um algoritmo. Cada variável possui um nome (identificador) e um tipo que define que tipo de dados ela pode armazenar.

Tipos de Dados Principais:

  • inteiro: Números inteiros (10, -5, 42)
  • real: Números decimais (3.14, -2.5)
  • logico: Valores booleanos (verdadeiro, falso)
  • caractere: Sequência de caracteres ("Olá, mundo!")

Declaração de Variáveis

var nome: caractere idade: inteiro altura: real estudante: logico

Regras para nomes de variáveis:

  • Devem ser significativos (evite: a, x, y, teste)
  • Usar padrão snake_case (nome_completo, idade_usuario)
  • Começar com letra ou underscore
  • Não podem ser palavras reservadas
  • Não podem conter caracteres especiais no meio

Operadores de Atribuição

Para atribuir valores às variáveis, usamos os operadores:

  • ← (seta para esquerda)
  • := (dois pontos igual)
  • = (igual simples)
nome ← "João" idade := 25 altura = 1.75

Aula 3: Estruturas de Controle

As estruturas de controle determinam o fluxo de execução de um algoritmo, permitindo decidir a ordem dos comandos.

3.1 Estrutura Sequencial

Executa comandos em ordem, um após o outro. É a estrutura mais básica.

leia(nome) escreva("Olá, ", nome) escreva("Bem-vindo ao curso!")

3.2 Estrutura Condicional Simples

Executa comandos apenas se a condição for verdadeira.

se numero > 0 então escreva("Número positivo") fimse

3.3 Estrutura Condicional Composta

Permite avaliar múltiplas condições com alternativas.

se numero > 0 então escreva("Positivo") senão se numero < 0 então escreva("Negativo") senão escreva("Zero") fimse fimse

Operadores Relacionais

Operadores para comparações:

  • > maior que
  • >= maior ou igual
  • < menor que
  • <= menor ou igual
  • == igual a
  • != diferente de

Exercício 4.4: Calculadora de IMC

  • IMC 25-29.9: "Sobrepeso"
  • IMC >= 30: "Obesidade"

Conceitos aplicados: Variáveis do tipo real, operações matemáticas, estruturas condicionais

💡 Solução do Exercício 4.4:

Explicação: Calculamos o IMC dividindo o peso pela altura ao quadrado, depois classificamos usando estruturas condicionais baseadas nos valores de referência médica.

  3. Defina as saídas esperadas (resultados a serem exibidos)

Aula 6: Estruturas de Repetição

O que são Estruturas de Repetição?

Imagine que você precise escrever os nomes de 50 alunos. Sem repetição, teria que digitar 50 vezes o comando leia e escreva. Com repetição, você escreve apenas uma vez e o computador repete o processo!

🔑 Definição: Estrutura de repetição é quando o algoritmo executa o mesmo conjunto de comandos várias vezes, até uma condição seja atendida.

6.1 Tipos de Estruturas de Repetição

📌 a) Enquanto / faça

Primeiro verifica a condição. Se for verdadeira, executa os comandos. Se for falsa, encerra.

algoritmo "exemplo_enquanto" var numero: inteiro inicio numero ← 1 enquanto numero <= 5 faça escreva(numero) numero ← numero + 1 fimenquanto Fimalgoritmo

👉 Resultado: Escreve os números 1, 2, 3, 4, 5.

📌 b) Repita / até

Primeiro executa os comandos, depois verifica a condição. Ou seja: sempre roda pelo menos uma vez.

algoritmo "exemplo_repita" var numero: inteiro inicio numero ← 1 repita escreva(numero) numero ← numero + 1 até numero > 5 Fimalgoritmo

👉 Resultado: Também escreve 1, 2, 3, 4, 5.

📌 c) Para / faça

Usado quando já sabemos quantas vezes vamos repetir. Bem mais simples!

algoritmo "exemplo_para" var numero: inteiro inicio para numero de 1 até 5 faça escreva(numero) fimpara Fimalgoritmo

👉 Resultado: Escreve 1, 2, 3, 4, 5.

6.2 Variáveis Importantes

🔹 Variável Contadora: Serve para controlar quantas vezes o laço foi repetido.

Exemplo: numero ← numero + 1

🔹 Variável Acumuladora: Serve para somar valores a cada repetição.

Exemplo: Somar notas, calcular total de faltas.

Exemplo Prático com Acumuladora:

algoritmo "calcula_media" var soma, nota, media: real cont: inteiro inicio soma ← 0 para cont de 1 até 3 faça escreva("Digite a nota ", cont, ": ") leia(nota) soma ← soma + nota fimpara media ← soma / 3 escreva("Média: ", media) Fimalgoritmo

6.3 Flag (Marcador)

Às vezes, não sabemos quantas vezes o laço vai repetir. Nesse caso, usamos uma palavra de parada (flag).

algoritmo "exemplo_flag" var pessoa: caractere inicio escreva("Digite um nome (ou 'FIM' para sair): ") leia(pessoa) enquanto pessoa <> "FIM" faça escreva("Olá, ", pessoa) escreva("Digite outro nome (ou 'FIM' para sair): ") leia(pessoa) fimenquanto escreva("Programa encerrado!") fim

👉 Aqui, o usuário digita nomes até escrever "FIM".

📚 Resumindo os tipos de repetição:

  • Enquanto: verifica antes de executar
  • Repita: executa pelo menos uma vez
  • Para: quando já sabemos quantas vezes repetir
  • Contadora: controla a quantidade de repetições
  • Acumuladora: soma valores ao longo do laço
  • Flag: palavra que indica quando parar

Aula 7: Vetores (Arrays)

O que são Vetores?

Imagine que você tem uma gaveta com várias divisórias numeradas, e em cada divisória você guarda um objeto. Um vetor é exatamente isso: uma "gaveta" na memória do computador com várias "caixinhas" numeradas onde você pode guardar valores!

🔑 Definição: Vetor é uma estrutura que armazena vários valores do mesmo tipo, organizados em sequência e acessados por um índice (número da posição).

Por que usar Vetores?

Sem vetor, para guardar 5 notas você precisaria de 5 variáveis:

nota1, nota2, nota3, nota4, nota5: real

Com vetor, você usa apenas uma variável:

notas: vetor[1..5] de real

7.1 Como Declarar um Vetor

algoritmo "exemplo_vetor" var numeros: vetor[1..5] de inteiro nomes: vetor[1..3] de caractere inicio // Aqui você pode usar os vetores fim

👉 Explicação:

  • numeros: vetor com 5 posições (1, 2, 3, 4, 5) para números inteiros
  • nomes: vetor com 3 posições (1, 2, 3) para textos

7.2 Como Usar um Vetor

📌 a) Atribuindo valores

algoritmo "preenchendo_vetor" var notas: vetor[1..3] de real inicio notas[1] ← 8.5 notas[2] ← 7.0 notas[3] ← 9.5 escreva("Primeira nota: ", notas[1]) escreva("Segunda nota: ", notas[2]) escreva("Terceira nota: ", notas[3]) fim

👉 Resultado: Mostra 8.5, 7.0 e 9.5

📌 b) Lendo valores do usuário

algoritmo "lendo_vetor" var idades: vetor[1..4] de inteiro i: inteiro inicio para i de 1 até 4 faça escreva("Digite a idade da pessoa ", i, ": ") leia(idades[i]) fimpara // Mostrando os valores escreva("Idades cadastradas:") para i de 1 até 4 faça escreva("Pessoa ", i, ": ", idades[i], " anos") fimpara fim

💡 Dica: Quase sempre usamos laços de repetição (para, enquanto) para trabalhar com vetores!

7.3 Operações Comuns com Vetores

🔹 Somar todos os valores:

soma ← 0 para i de 1 até 5 faça soma ← soma + vetor[i] fimpara

🔹 Encontrar o maior valor:

maior ← vetor[1] para i de 2 até 5 faça se vetor[i] > maior então maior ← vetor[i] fimse fimpara

🔹 Buscar um valor específico:

encontrou ← falso para i de 1 até 5 faça se vetor[i] = valorProcurado então encontrou ← verdadeiro fimse fimpara

Exemplo Prático Completo: Sistema de Notas

algoritmo "sistema_notas" var notas: vetor[1..5] de real soma, media: real i: inteiro inicio // Lendo as notas escreva("=== CADASTRO DE NOTAS ===") para i de 1 até 5 faça escreva("Digite a nota ", i, ": ") leia(notas[i]) fimpara // Calculando a média soma ← 0 para i de 1 até 5 faça soma ← soma + notas[i] fimpara media ← soma / 5 // Mostrando resultado escreva("Notas cadastradas:") para i de 1 até 5 faça escreva("Nota ", i, ": ", notas[i]) fimpara escreva("Média final: ", media) se media >= 7 então escreva("Aluno APROVADO!") senão escreva("Aluno REPROVADO!") fimse fim

📚 Resumindo Vetores:

  • Vetor: guarda vários valores do mesmo tipo
  • Índice: número da posição (começa em 1 no Visualg)
  • Declaração: nome: vetor[inicio..fim] de tipo
  • Acesso: nome[posição] para ler ou escrever
  • Laços: use "para" para percorrer todo o vetor
  • Tamanho fixo: defina o tamanho na declaração

Exercício 7.1: Vetor Inverso

Crie um algoritmo que leia 6 números inteiros e os armazene em um vetor. Depois, mostre os números na ordem inversa (do último para o primeiro).

Exercício 7.2: Maior e Menor Temperatura

Faça um algoritmo que leia as temperaturas dos 7 dias da semana e armazene em um vetor. Depois, mostre qual foi a maior e a menor temperatura da semana.

Conceitos aplicados: Vetores, busca de maior e menor valor

💡 Solução do Exercício 7.2:
algoritmo "temperaturas" var temp: vetor[1..7] de real maior, menor: real i: inteiro inicio escreva("=== TEMPERATURAS DA SEMANA ===") para i de 1 até 7 faça escreva("Temperatura do dia ", i, ": ") leia(temp[i]) fimpara // Inicializa maior e menor com o primeiro valor maior ← temp[1] menor ← temp[1] // Busca o maior e menor para i de 2 até 7 faça se temp[i] > maior então maior ← temp[i] fimse se temp[i] < menor então menor ← temp[i] fimse fimpara escreva("Maior temperatura: ", maior, "°C") escreva("Menor temperatura: ", menor, "°C") fim

Explicação: Guardamos todas as temperaturas no vetor. Depois, comparamos cada valor para encontrar o maior e o menor.

Exercício 7.3: Buscar Nome na Lista

Desenvolva um algoritmo que cadastre 5 nomes em um vetor. Depois, peça um nome para buscar e informe se ele está ou não na lista.

Conceitos aplicados: Vetores de caracteres, busca, flag

💡 Solução do Exercício 7.3:
algoritmo "buscar_nome" var nomes: vetor[1..5] de caractere busca: caractere encontrou: logico i: inteiro inicio escreva("=== CADASTRO DE NOMES ===") para i de 1 até 5 faça escreva("Digite o nome ", i, ": ") leia(nomes[i]) fimpara escreva("Digite o nome que deseja buscar: ") leia(busca) encontrou ← falso para i de 1 até 5 faça se nomes[i] = busca então encontrou ← verdadeiro fimse fimpara se encontrou então escreva("O nome ", busca, " ESTÁ na lista!") senão escreva("O nome ", busca, " NÃO está na lista!") fimse fim

Explicação: Usamos uma variável lógica (encontrou) como flag para indicar se o nome foi encontrado durante a busca no vetor.

Exercício 7.4: Média e Alunos Acima da Média

Crie um programa que leia as notas de 8 alunos, calcule a média da turma e mostre quantos alunos ficaram acima da média.

Conceitos aplicados: Vetores, média, contador, múltiplos laços

💡 Solução do Exercício 7.4:
algoritmo "media_turma" var notas: vetor[1..8] de real soma, media: real i, acimaDaMedia: inteiro inicio escreva("=== NOTAS DA TURMA ===") para i de 1 até 8 faça escreva("Digite a nota do aluno ", i, ": ") leia(notas[i]) fimpara // Calculando a média soma ← 0 para i de 1 até 8 faça soma ← soma + notas[i] fimpara media ← soma / 8 // Contando quantos ficaram acima da média acimaDaMedia ← 0 para i de 1 até 8 faça se notas[i] > media então acimaDaMedia ← acimaDaMedia + 1 fimse fimpara escreva("Média da turma: ", media) escreva("Alunos acima da média: ", acimaDaMedia) fim

Explicação: Primeiro calculamos a média de todos. Depois, percorremos o vetor novamente para contar quantos alunos têm nota maior que a média.

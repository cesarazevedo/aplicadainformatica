Aula 1: Introdução aos Algoritmos

O que são Algoritmos?

Um algoritmo é uma sequência de instruções realizadas com o objetivo de solucionar um problema. Em programação, utilizamos algoritmos para descrever as etapas que precisam ser efetuadas para que um programa execute as tarefas que lhe são designadas.

Características dos Algoritmos:

  • Conjunto de passos finitos e organizados
  • Quando executados, resolvem um determinado problema
  • São compostos por entrada (input), processamento e saída (output)

Pseudocódigo (Portugol)

O pseudocódigo é um método genérico de representar um algoritmo, usando uma linguagem simples sem necessidade de conhecer a sintaxe de linguagem de programação específica.

algoritmo "exemplo_basico" var numero: inteiro dobro: inteiro inicio leia(numero) dobro ← numero * 2 escreva("O dobro é: ", dobro) fim

Estrutura Básica de um Algoritmo

Todo algoritmo em pseudocódigo segue uma estrutura padrão:

algoritmo "nome_do_algoritmo" // comentários e informações do autor var // declaração de variáveis inicio // comandos do programa fim

Aula 2: Variáveis e Tipos de Dados

O que são Variáveis?

Variáveis são espaços alocados na memória do computador para armazenar dados ou expressões utilizados por um algoritmo. Cada variável possui um nome (identificador) e um tipo que define que tipo de dados ela pode armazenar.

Tipos de Dados Principais:

  • inteiro: Números inteiros (10, -5, 42)
  • real: Números decimais (3.14, -2.5)
  • logico: Valores booleanos (verdadeiro, falso)
  • caractere: Sequência de caracteres ("Olá, mundo!")

Declaração de Variáveis

var nome: caractere idade: inteiro altura: real estudante: logico

Regras para nomes de variáveis:

  • Devem ser significativos (evite: a, x, y, teste)
  • Usar padrão snake_case (nome_completo, idade_usuario)
  • Começar com letra ou underscore
  • Não podem ser palavras reservadas
  • Não podem conter caracteres especiais no meio

Operadores de Atribuição

Para atribuir valores às variáveis, usamos os operadores:

  • ← (seta para esquerda)
  • := (dois pontos igual)
  • = (igual simples)
nome ← "João" idade := 25 altura = 1.75

Aula 3: Estruturas de Controle

As estruturas de controle determinam o fluxo de execução de um algoritmo, permitindo decidir a ordem dos comandos.

3.1 Estrutura Sequencial

Executa comandos em ordem, um após o outro. É a estrutura mais básica.

leia(nome) escreva("Olá, ", nome) escreva("Bem-vindo ao curso!")

3.2 Estrutura Condicional Simples

Executa comandos apenas se a condição for verdadeira.

se numero > 0 então escreva("Número positivo") fimse

3.3 Estrutura Condicional Composta

Permite avaliar múltiplas condições com alternativas.

se numero > 0 então escreva("Positivo") senão se numero < 0 então escreva("Negativo") senão escreva("Zero") fimse

Operadores Relacionais

Operadores para comparações:

  • > maior que
  • >= maior ou igual
  • < menor que
  • <= menor ou igual
  • == igual a
  • != diferente de

Exercício 4.1: Calculadora de Idade

Escreva um algoritmo em pseudocódigo que leia o ano de nascimento de uma pessoa e calcule e imprima a idade dela (considere o ano atual como 2025).

Conceitos aplicados: Variáveis, operações aritméticas, entrada e saída de dados

💡 Solução do Exercício 4.1:
algoritmo "calcula_idade" var ano_nascimento: inteiro idade: inteiro inicio escreva("Digite seu ano de nascimento: ") leia(ano_nascimento) idade ← 2025 - ano_nascimento escreva("Sua idade é: ", idade) fim

Explicação: Realizamos uma subtração simples entre o ano atual (2025) e o ano de nascimento informado pelo usuário.

Exercício 4.2: Verificação de Número Par

Desenvolva um algoritmo que leia um número inteiro e verifique se ele é par. Se for par, exiba "O número é par", caso contrário, não exiba nada.

Conceitos aplicados: Estrutura condicional simples, operador módulo (%)

💡 Solução do Exercício 4.2:
algoritmo "verifica_par" var numero: inteiro inicio escreva("Digite um número: ") leia(numero) se numero % 2 = 0 então escreva("O número é par") fimse fim

Explicação: Usamos o operador módulo (%) para verificar o resto da divisão por 2. Se o resto for 0, o número é par.

Exercício 4.3: Classificação de Nota

Crie um algoritmo que leia uma nota (0 a 10) e classifique conforme os critérios:

  • Nota >= 9: "Excelente"
  • Nota >= 7: "Bom"
  • Nota >= 5: "Regular"
  • Nota < 5: "Insuficiente"

Conceitos aplicados: Estrutura condicional composta, operadores relacionais

💡 Solução do Exercício 4.3:
algoritmo "classifica_nota" var nota: real inicio escreva("Digite a nota (0-10): ") leia(nota) se nota >= 9 então escreva("Excelente") senão se nota >= 7 então escreva("Bom") senão se nota >= 5 então escreva("Regular") senão escreva("Insuficiente") fimse fim

Explicação: Utilizamos estrutura condicional composta (se-senão se-senão) para testar múltiplas condições em ordem decrescente.

Exercício 4.4: Calculadora de IMC

Desenvolva um algoritmo que calcule o IMC (peso / altura²) e classifique o resultado:

  • IMC < 18.5: "Abaixo do peso"
  • IMC 18.5-24.9: "Peso normal"
  • IMC 25-29.9: "Sobrepeso"
  • IMC >= 30: "Obesidade"

Conceitos aplicados: Variáveis do tipo real, operações matemáticas, estruturas condicionais

💡 Solução do Exercício 4.4:
algoritmo "calcula_imc" var peso, altura, imc: real inicio escreva("Digite seu peso (kg): ") leia(peso) escreva("Digite sua altura (m): ") leia(altura) imc ← peso / (altura * altura) escreva("Seu IMC é: ", imc) se imc < 18.5 então escreva("Abaixo do peso") senão se imc <= 24.9 então escreva("Peso normal") senão se imc <= 29.9 então escreva("Sobrepeso") senão escreva("Obesidade") fimse fim

Explicação: Calculamos o IMC dividindo o peso pela altura ao quadrado, depois classificamos usando estruturas condicionais baseadas nos valores de referência médica.

Dica para resolver os exercícios:

  1. Identifique as entradas necessárias (dados que o usuário deve fornecer)
  2. Determine o processamento (cálculos e lógica a serem aplicados)
  3. Defina as saídas esperadas (resultados a serem exibidos)
  4. Escolha a estrutura de controle adequada (sequencial, condicional simples ou composta)

Aula 6: Estruturas de Repetição

O que são Estruturas de Repetição?

Imagine que você precise escrever os nomes de 50 alunos. Sem repetição, teria que digitar 50 vezes o comando leia e escreva. Com repetição, você escreve apenas uma vez e o computador repete o processo!

🔑 Definição: Estrutura de repetição é quando o algoritmo executa o mesmo conjunto de comandos várias vezes, até que uma condição seja atendida.

6.1 Tipos de Estruturas de Repetição

📌 a) Enquanto / faça

Primeiro verifica a condição. Se for verdadeira, executa os comandos. Se for falsa, encerra.

algoritmo "exemplo_enquanto" var numero: inteiro inicio numero ← 1 enquanto numero <= 5 faça escreva(numero) numero ← numero + 1 fimenquanto fim

👉 Resultado: Escreve os números 1, 2, 3, 4, 5.

📌 b) Repita / até que

Primeiro executa os comandos, depois verifica a condição. Ou seja: sempre roda pelo menos uma vez.

algoritmo "exemplo_repita" var numero: inteiro inicio numero ← 1 repita escreva(numero) numero ← numero + 1 até que numero > 5 fim

👉 Resultado: Também escreve 1, 2, 3, 4, 5.

📌 c) Para / faça

Usado quando já sabemos quantas vezes vamos repetir. Bem mais simples!

algoritmo "exemplo_para" var numero: inteiro inicio para numero de 1 até 5 faça escreva(numero) fim-para fim

👉 Resultado: Escreve 1, 2, 3, 4, 5.

6.2 Variáveis Importantes

🔹 Variável Contadora: Serve para controlar quantas vezes o laço foi repetido.

Exemplo: numero ← numero + 1

🔹 Variável Acumuladora: Serve para somar valores a cada repetição.

Exemplo: Somar notas, calcular total de faltas.

Exemplo Prático com Acumuladora:

algoritmo "calcula_media" var soma, nota, media: real cont: inteiro inicio soma ← 0 para cont de 1 até 3 faça escreva("Digite a nota ", cont, ": ") leia(nota) soma ← soma + nota fim-para media ← soma / 3 escreva("Média: ", media) fim

6.3 Flag (Marcador)

Às vezes, não sabemos quantas vezes o laço vai repetir. Nesse caso, usamos uma palavra de parada (flag).

algoritmo "exemplo_flag" var pessoa: caractere inicio escreva("Digite um nome (ou 'FIM' para sair): ") leia(pessoa) enquanto pessoa <> "FIM" faça escreva("Olá, ", pessoa) escreva("Digite outro nome (ou 'FIM' para sair): ") leia(pessoa) fimenquanto escreva("Programa encerrado!") fim

👉 Aqui, o usuário digita nomes até escrever "FIM".

📚 Resumindo os tipos de repetição:

  • Enquanto: verifica antes de executar
  • Repita: executa pelo menos uma vez
  • Para: quando já sabemos quantas vezes repetir
  • Contadora: controla a quantidade de repetições
  • Acumuladora: soma valores ao longo do laço
  • Flag: palavra que indica quando parar
